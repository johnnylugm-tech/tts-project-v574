## 1. 簡介

### 1.1 目的

本文件旨在定義基於 Microsoft Edge TTS 技術的簡報配音系統的功能性、非功能性需求以及技術約束。系統專為台灣市場設計，提供高品質、自然流暢的台灣國語語音合成服務。

### 1.2 範圍

本系統提供以下核心功能：
- 基於 edge-tts 的文字轉語音合成
- 台灣國語曉曉音色（zh-TW-HsiaoHsiaoNeural）
- 智能文本分段處理
- WebSocket 非同步通訊
- 語速與音量參數控制
- 錯誤處理與重試機制

### 1.3 目標使用者

- 簡報製作者
- 教育訓練人員
- 內容創作者
- 需要台灣國語配音的專業人士

## 2. 功能需求

### 2.1 語音合成引擎

| ID | 需求 | 優先級 |
|----|------|--------|
| FR-01 | 系統必須能夠透過 edge-tts 將文字轉換為語音 | P0 |
| FR-02 | 系統必須預設使用 zh-TW-HsiaoHsiaoNeural 音色 | P0 |
| FR-03 | 系統必須支援自訂音色選擇 | P1 |
| FR-04 | 系統必須輸出 MP3 格式音訊檔案 | P0 |

### 2.2 文本處理

| ID | 需求 | 優先級 |
|----|------|--------|
| FR-05 | 系統必須能夠根據中文句號（。）、問號（？）、驚嘆號（！）進行智能分段 | P0 |
| FR-06 | 系統必須能夠根據換行符（\n）進行分段 | P0 |
| FR-07 | 系統必須限制單次分段長度不超過 800 字（可配置） | P0 |
| FR-08 | 系統必須支持最大 20,000 字的單次輸入 | P1 |

### 2.3 參數控制

| ID | 需求 | 優先級 |
|----|------|--------|
| FR-09 | 系統必須支援語速（rate）參數調整，範圍為 -100% 至 +100% | P0 |
| FR-10 | 系統必須支援音量（volume）參數調整，範圍為 -100% 至 +100% | P1 |
| FR-11 | 系統預設語速為 -2%（略慢，適合簡報） | P0 |
| FR-12 | 系統預設音量為 +0%（標準） | P1 |

### 2.4 通訊機制

| ID | 需求 | 優先級 |
|----|------|--------|
| FR-13 | 系統必須使用 WebSocket 與 edge-tts 伺服器通訊 | P0 |
| FR-14 | 系統必須支援流式（Streaming）音訊回傳 | P0 |
| FR-15 | 系統必須基於 asyncio 框架實現非同步處理 | P0 |

### 2.5 錯誤處理

| ID | 需求 | 優先級 |
|----|------|--------|
| FR-16 | 系統必須實現網路錯誤重試機制（預設 3 次） | P0 |
| FR-17 | 系統必須實現熔斷器（Circuit Breaker）機制 | P1 |
| FR-18 | 系統必須對不同錯誤類型進行分類處理（L1-L4） | P1 |

### 2.6 音訊輸出

| ID | 需求 | 優先級 |
|----|------|--------|
| FR-19 | 系統必須能夠將多個分段音訊合併為單一輸出檔案 | P0 |
| FR-20 | 系統必須使用 ffmpeg 進行音訊合併 | P1 |

## 3. 非功能性需求

### 3.1 效能需求

| ID | 需求 | 目標 |
|----|------|------|
| NFR-01 | 首位元組時間（Time to First Byte） | < 2 秒 |
| NFR-02 | 單次合成最大延遲（10,000 字） | < 60 秒 |
| NFR-03 | 並發處理能力 | 支援多使用者同時請求 |

### 3.2 可靠性需求

| ID | 需求 | 目標 |
|----|------|------|
| NFR-04 | 系統正常運作率 | 99.5% |
| NFR-05 | 錯誤自動恢復率 | > 90% |

### 3.3 安全性需求

| ID | 需求 |
|----|------|
| NFR-06 | 不儲存用戶敏感文字資訊 |
| NFR-07 | API 請求記錄日誌管理 |

### 3.4 相容性需求

| ID | 需求 |
|----|------|
| NFR-08 | Python 3.8+ 相容 |
| NFR-09 | 支援主流作業系統（Windows、macOS、Linux）|

## 4. 技術架構

### 4.1 系統架構圖

```
┌─────────────────────────────────────────────────────────┐
│                    接入通訊層                             │
│  ┌───────────────────────────────────────────────────┐  │
│  │            WebSocket 非同步請求                    │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│                   文本分析與處理層                        │
│  ┌───────────────────────────────────────────────────┐  │
│  │     智能分段（句號、問號、驚嘆號、換行符）          │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│                    合成引擎層                             │
│  ┌───────────────────────────────────────────────────┐  │
│  │  edge-tts Communicate (音色、語速、音量)          │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│                  後處理與匯出層                           │
│  ┌───────────────────────────────────────────────────┐  │
│  │     音訊合併（ffmpeg）、輸出 MP3                   │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 4.2 核心模組設計

| 模組 | 職責 | 對應檔案 |
|------|------|----------|
| ConfigManager | 參數管理（frozen dataclass） | config_manager.py |
| TextProcessor | 文本分段處理 | text_processor.py |
| AsyncSynthesizer | 非同步語音合成 | async_synthesizer.py |
| AudioMerger | 音訊合併 | audio_merger.py |
| ErrorHandler | 錯誤處理與分類 | error_handler.py |

## 5. 介面定義

### 5.1 程式化介面

```python
class PresentationTTS:
    def __init__(
        self,
        voice: str = "zh-TW-HsiaoHsiaoNeural",
        rate: str = "-2%",
        volume: str = "+0%"
    ): ...
    
    async def synthesize(self, text: str, output_file: str) -> str: ...
    async def synthesize_batch(self, texts: List[str], output_dir: str) -> List[str]: ...
```

### 5.2 CLI 介面

```bash
python tts_cli.py --text "簡報內容" --output output.mp3
python tts_cli.py --file input.txt --output output.mp3
python tts_cli.py --rate "-2%" --volume "+0%" --voice "zh-TW-HsiaoHsiaoNeural"
```

## 7. 風險與假設

### 7.1 風險

| 風險 | 影響 | 緩解措施 |
|------|------|----------|
| 網路不穩定 | 合成失敗 | 重試機制 + 熔斷器 |
| API 配額限制 | 服務中斷 | 速率限制 + 錯誤訊息 |

### 7.2 假設

- 用戶網路可正常存取 Microsoft edge-tts 伺服器
- 用戶已安裝 ffmpeg（用於音訊合併）
- 輸入文字為 UTF-8 編碼
